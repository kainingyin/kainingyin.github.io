4.Flux

Flux 是一种架构思想，专门解决软件的结构问题。它跟MVC 架构是同一类东西，但是更加简单和清晰。

Flux将一个应用分成四个部分。

  • View： 视图层
  • Action（动作）：视图层发出的消息（比如mouseClick）
  • Dispatcher（派发器）：用来接收Actions、执行回调函数
  • Store（数据层）：用来存放应用的状态，一旦发生变动，就提醒Views要更新页面

Flux 的最大特点，就是数据的”单向流动”。

  1. 用户访问 View
  2. View 发出用户的 Action
  3. Dispatcher 收到 Action，要求 Store 进行相应的更新
  4. Store 更新后，发出一个”change”事件
  5. View 收到”change”事件后，更新页面

5.Redux

1.Redux介绍

Redux是Flux的一种实现。Redux是SPA单页面应用程序中多个组件之间共享数据的一种方式。

Redux在Flux基础上强调三个基本原则：

    1. 数据源唯一

    2. 不允许随便修改store数据

    3. store数据的修改需要通过纯函数来完成 Reducer

关于冒泡排序

定义：比较相邻的两个元素

比较相邻的两个元素，如果第一个元素比第二个元素大，则交换位置。对每一对元素都进行比较，直到没有任何数需要比较为止。

原理：如 要求一个数组需要从小到大排序

如 要求一个数组需要从小到大排序

var arr = [3,46,19,5,48];

第一次大循环，四次小循环：原数组是3,46,19,5,48; 

比较3和46         ==>        3,46,19,5,48;            

比较46和19       ==>      3,19,46,5,48;    

比较46和5         ==>      3,19,5,46,48;

比较46和48       ==>      3,19,5,46,48;

第二次大循环，三次小循环：原数组是3,19,5,46 ;//此时48已是最大数所以不用再比较 

​比较3和19         ==>        3,19,5,46;            

比较19和5         ==>      3,5,19,46;    

比较19和46       ==>      3,5,19,46;

第三次大循环，二次小循环：原数组是3,19,5;//此时46已是次大数所以不用再比较 

​比较3和19         ==>        3,19,5;            

比较19和5         ==>      3,5,19;

第四次大循环，一次小循环：原数组是3,5;//此时19已是次次大数所以不用再比较 

​比较3和5         ==>        3,5;
